# nww-home

Dual-brand static marketing site built with Next.js 16 App Router.

- **`/`** — [NarenWebWorks](https://www.narenwebworks.com) — freelance software consultancy
- **`/clubcaddy`** — [ClubCaddy](https://www.narenwebworks.in/clubcaddy/) — golf club management SaaS

## Tech Stack

- **Next.js 16** (App Router, fully static output)
- **TypeScript**
- **Tailwind CSS**
- **Google Fonts** — Montserrat + Source Sans 3

## Project Structure

```
src/
├── app/
│   ├── page.tsx              # NWW home (/)
│   ├── layout.tsx            # Root layout + shared metadata
│   ├── globals.css           # CSS custom properties + dual-brand theme
│   ├── robots.ts             # /robots.txt
│   ├── sitemap.ts            # /sitemap.xml
│   └── clubcaddy/
│       ├── page.tsx          # ClubCaddy landing (/clubcaddy)
│       └── layout.tsx        # ClubCaddy layout + metadata
├── components/
│   ├── nww/                  # NWW-specific page sections
│   ├── sections/             # ClubCaddy-specific page sections
│   ├── layout/               # Navbar, Footer (per brand)
│   ├── shared/               # AnimatedElement, Logo, SectionWrapper
│   └── ui/                   # Button, Card, Input, Toast, etc.
├── data/                     # Features, pricing, navigation content
├── hooks/                    # useInView, useReducedMotion, useScrollPosition
└── lib/
    ├── constants.ts          # NWW_CONFIG, CLUBCADDY_CONFIG, colors, nav links
    ├── schema.ts             # JSON-LD structured data (NWW + ClubCaddy)
    └── metadata.ts           # (removed — was dead code)
```

## Theming

The two brands share a single CSS custom-property system in `globals.css`. Adding the `.theme-nww` or `.theme-clubcaddy` class to a parent element switches all `--color-*` variables:

| Token | NWW | ClubCaddy |
|---|---|---|
| `--color-primary` | `#B163FF` (violet) | `#01941a` (green) |

Components reference `var(--color-primary)` — the active theme class determines which brand's values apply.

## Commands

```bash
npm run dev      # Start development server (http://localhost:3000)
npm run build    # Production build + TypeScript type checking
npm run start    # Start production server
npm run lint     # Run ESLint
```

No test runner is configured.

## Content

All copy, URLs, and contact details live in `src/lib/constants.ts` and `src/data/`. Edit those files first when updating brand information — most components pull from them rather than hardcoding values.

## SEO

- Page-level metadata is defined in each route's `layout.tsx` / `page.tsx`
- JSON-LD structured data is injected directly in `src/app/page.tsx` (NWW `ProfessionalService` schema) and `src/app/clubcaddy/page.tsx` (ClubCaddy `SoftwareApplication` schema) via `src/lib/schema.ts`
- Sitemap: `/sitemap.xml` — generated by `src/app/sitemap.ts`
- Robots: `/robots.txt` — generated by `src/app/robots.ts`
